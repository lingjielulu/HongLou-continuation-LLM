## 一、问题：模型为什么会"复活"已死角色

### 当前流程

```
大纲（本回情节）
    +
前文结尾（上一回末尾约400字）
    ↓
write_chapter_instruct.py → 生成章回
```

### 问题根源

训练阶段以 **n-gram overlap** 和 **perplexity** 作为核心指标，这两个指标都奖励"贴近训练语料"的输出。结果是：

- 模型对原文 80 回的人物状态有极强的隐式记忆
- 生成时倾向于复现训练语料中出现频率高的人物组合与场景
- 当 prompt 中没有明确的"世界状态"信息时，模型会自动回落到原文分布

### 问题的本质

当前 prompt 只有两层信息：**将发生什么**（大纲）+ **刚刚发生了什么**（前文结尾）。

缺失的是第三层：**到目前为止，世界是什么状态**。

### 两层解法

解法分为两个独立层次，各自有清晰的职责边界：

```
┌─────────────────────────────────────────┐
│  Prompt 层（显式上下文注入）              │
│  · 年表：截至本回前发生了什么            │
│  · 人物状态卡：各人物当前是什么状态      │
│  职责：在推理时约束生成空间              │
└─────────────────────────────────────────┘
                    ↕  相互独立，可分别迭代
┌─────────────────────────────────────────┐
│  训练层（隐式权重与奖励信号）             │
│  · 对齐指标：量化生成与年表/状态卡的偏差 │
│  · 训练方法：将偏差信号接入训练目标      │
│  职责：从权重层面降低违规倾向            │
└─────────────────────────────────────────┘
```

两层解决不同的问题：Prompt 层通过显式信息限制生成空间；训练层通过改变模型权重降低违规的先验概率。单独依赖任意一层都不够。

# Part I：Prompt 层

## 二、年表（Timeline）

### 体例来源

年表体例源自司马迁《史记》的《十二诸侯年表》，后世正史沿用，属"表谱"类工具书。续写系统的年表最接近**大事年表与个人生平年表的结合体**：以回目为时间单位，记录关键事件及其对人物的影响。

### 字段设计

| 字段 | 说明 |
|---|---|
| 回目 | 替代传统年表的"年代"，以章回为时间单位 |
| 叙事纪时 | 小说内的时间表述（如"时值深秋"）；《红楼梦》有意模糊朝代，不强制标注年号 |
| 事件摘要 | 30字以内，只陈述事实，不发议论 |
| 影响人物 | 人名列表，用于触发人物状态卡的更新 |
| 事件性质 | 标注"不可逆"或"可逆"，决定后续生成能否撤销该事件 |

### 事件粒度：只记录不可逆的状态变更

筛选原则是**不可逆性**——发生后无法撤销、且影响后续叙事的事件才进入年表。

**进入年表：**
- 人物死亡、远嫁、出家（身份与位置永久改变）
- 重大财产或权力转移（抄家、赐婚、贬谪）
- 关键物品的毁坏或赠予（通灵宝玉丢失、黛玉焚稿）

**不进入年表（交给上文层）：**
- 日常对话、情绪波动、短途移动

### 注入方式

生成第 N 回时，把第 81 回到第 N-1 回的年表注入 prompt。

## 三、人物状态卡（Persona）

> 人物 cards 存放在仓库 `../Hongloumeng_card/cards/`（每人一份 `*.md`），可直接作为 persona 的数据源或模板。

### 体例来源

融合两种传统：
- **历史人物小传**：言简意赅陈述生平，强调"用事实说话，不发议论"。提供客观固定属性。
- **小说/剧本人物设定**：侧重性格、动机、命运走向等动态叙事要素。

结合 CBDB（中国历代人物传记资料库）的结构化思路：**将人物视为实体，通过字段记录各维度属性**。

### 静态字段（一次性建立）

| 字段 | 说明 |
|---|---|
| 姓名、字号 | 含别称、昵称 |
| 出身 | 籍贯、家族门第 |
| 初始身份 | 第80回末的角色定位 |
| 初始关系 | 与主要人物的初始关系 |
| 性格特征 | 核心性格词，用于约束行为与对话风格 |

### 动态字段（每回更新）

| 字段 | 说明 |
|---|---|
| 当前状态 | 在世 / 已逝 / 下落不明 |
| 所在位置 | 宏观位置（京城/金陵）+ 微观锚点（潇湘馆/王夫人房中） |
| 当前处境与情感 | 简短描述当前生活处境与情绪状态 |
| 关键关系变化 | 自上次更新以来的重要关系变动 |
| 最后出场回目 | 最近一次作为行为主体出现的回目 |
| **允许出现形式** | 约束该人物在文中可以以何种形式出现（见下） |

**"允许出现形式"**是传统格式中没有的字段，专为约束生成而设：

- `主动`：可发起对话、执行动作（在世人物）
- `被动提及`：可被他人以称谓回忆，但无任何主动行为
- `意象出现`：只能出现在诗词、祭文、旧物联想中，不得有言语或行为描写

### 位置字段的两层设计

- **宏观位置**：京城 / 金陵 / 铁槛寺等
- **微观锚点**：当前正身处的具体场景

规则：若本回需要人物跨宏观位置移动，必须在前文或本回开头有明确的移动交代。

### 注入方式

只注入本回**出场人物**的状态卡。

**例外**：若大纲标记为"集中悼念/回忆某已逝人物"，则主动注入该人物状态卡，防止在悼念段落误写其复活。

## 四、年表与状态卡的一致性校验

以**年表为准**。每次更新状态卡时，检查年表中最新一条涉及该人物的不可逆事件是否与状态卡字段一致。若不一致，强制以年表覆盖并告警。

此校验在生成前的数据准备阶段完成，不依赖模型。

---

## 五、约束示例：林黛玉

```
林黛玉
  当前状态：已逝（第89-90回焚稿泪尽而逝）
  允许出现形式：
    - 不可作为对话发起者或动作执行者
    - 可在其他人物台词中以"林妹妹"、"颦儿"等称谓被回忆
    - 可出现在诗词、祭文、旧物引发的联想中，但不得有新的言语或行为描写
```

# Part II：训练层

## 六、对齐指标

对齐指标的目标是量化生成文本与年表/状态卡的偏差程度。共两组：timeline 对齐和 persona 对齐。所有指标均在纯文本空间计算。

### Timeline 对齐指标

**1. 世界状态一致性分（World State Consistency, WSC）**

```
WSC = 生成文本中，年表状态被正确反映的实体-状态对数
      ÷ 生成文本中涉及的年表实体-状态对总数
```

年表中的"状态"不限于生死，还包括：人物位置、身份变化（出家/远嫁）、权力关系（抄家后的贾府）、关键物品归属。WSC 衡量的是生成文本对整个年表世界状态的遵循程度。

**2. 事件引用准确率（Event Reference Accuracy, ERA）**

```
ERA = 生成文本中对年表事件的正确引用次数
      ÷ 生成文本中对年表事件的总引用次数
```

当生成文本显式回忆或提及过去事件时（"自探春远嫁后"、"想起黛玉焚稿之事"），检查该引用与年表记录是否一致。ERA 捕捉的是模型对已发生事件的"记忆准确性"，而非是否提及。

**3. 时序一致率（Temporal Order Consistency, TOC）**

```
TOC = 生成文本中事件对的时序与年表一致的比率
```

当生成文本暗示两个年表事件有先后关系时，检查该顺序是否与年表一致。

---

### Persona 对齐指标

**4. 人物文本风格相似度（Character Text Style Similarity, CTSS）**

```
CTSS(角色 X) = ngram_overlap(
    生成文本中 X 的对话与行为描写,
    原著前80回中 X 的对话与行为描写
)
```

与全局 n-gram overlap 不同，CTSS 是**按角色切分**的，衡量的是各角色自身的文本风格一致性（用词习惯、句式偏好、情感表达方式）而非整体文风。宝玉的文本片段应接近原著宝玉，黛玉的应接近原著黛玉，而非笼统地"接近原著"。

**5. 人物属性覆盖率（Character Attribute Coverage, CAC）**

```
CAC = 生成文本中正确反映的状态卡属性数
      ÷ 生成文本中理应涉及的状态卡属性总数
```

状态卡中每个动态字段（位置、处境、关系变化等）都是一个可检查的属性。当生成文本涉及某属性所在的叙事维度时，该属性应被正确体现。CAC 衡量的是状态卡各维度的整体遵循程度，而非单一维度（如生死）。

**6. 人际关系一致率（Relationship Consistency Rate, RCR）**

```
RCR = 生成文本中，人物互动与状态卡关系字段一致的互动对数
      ÷ 生成文本中涉及的人物关系对总数
```

人物关系在续写过程中会发生变化（如宝黛关系随黛玉死亡而终结，宝钗与宝玉的关系进入新阶段）。RCR 检查生成文本中的人物互动是否与状态卡中当前关系状态相符。

---

### 指标汇总

| 指标 | 缩写 | 对齐对象 | 计算层 | 集成时机 |
|---|---|---|---|---|
| 世界状态一致性分 | WSC | Timeline | 规则 + 模板 | 每个 checkpoint |
| 事件引用准确率 | ERA | Timeline | 规则 | 每 25 步 |
| 时序一致率 | TOC | Timeline | 规则 | 每个 checkpoint |
| 人物文本风格相似度 | CTSS | Persona | n-gram | 每个 checkpoint |
| 人物属性覆盖率 | CAC | Persona | 规则 | 每 25 步 |
| 人际关系一致率 | RCR | Persona | 规则 | 每个 checkpoint |

## 七、将对齐指标接入训练

**路径 A：DPO 负样本自动构造（近期可做）**

利用规则型指标（ERA、CAC）自动生成偏好对：
- 对同一 prompt 生成多个候选章回
- 指标低的候选标记为 `rejected`
- 指标高的候选标记为 `chosen`
- 在这批偏好对上做 DPO 微调

负样本来自模型自己的低对齐生成，构造成本低，不需要人工标注。

**路径 B：课程式约束收紧（中期）**

从易到难引入约束：先让 ERA → 1（事件引用准确），再优化 CAC（属性覆盖），最后优化 CTSS（细粒度风格一致）。每阶段只用当前最难通过的指标构造 DPO 数据，避免训练信号混淆。

**路径 C：n-gram 评估解耦（长期）**

当前 n-gram overlap 无法区分"风格记忆"和"内容记忆"。长期将 CTSS 按角色拆分，替代全局 n-gram 作为风格指标，同时引入 WSC/ERA 作为内容约束指标，两类指标分别优化。
