# 文献综述：详细评注与对比

> 本文档是 `project_proposal.md` 的**支撑文档**。提案第三节（相关工作）已包含各方向的批判性综述和总体研究空白分析。
> 本文件按单篇论文提供详细的要点提取、项目关联，以及提案中不便展开的逐条对比。

---

## 一、LLM 叙事生成

### PlotMachines (Rashkin et al., 2020) — EMNLP
- **核心**：大纲短语（outline phrases）控制故事走向，动态情节状态追踪
- **与项目关联**：证明了 outline-conditioned generation 可行性，本项目大纲层的设计受其启发
- **差异**：PlotMachines 评估仅限短篇，无跨章节状态一致性需求

### DOC (Yang et al., 2022) — ACL
- **核心**：plan → draft → polish 三阶段生成，提升中篇故事连贯性
- **与项目关联**：分阶段策略与本项目三层架构（大纲→状态→表达）对应
- **差异**：DOC 的 plan 层仅含情节节点，不包含世界状态变更（人物生死/位置等）

### Re3 (Yang et al., 2023) — EMNLP
- **核心**：recall → relate → revise 三步提升叙事一致性
- **差异**：依赖预训练重排序器，不适用于古典文学（无人工标注数据）

### FableStudio (Majumder et al., 2024)
- **核心**：多 Agent（作者、角色、评论员）协作生成寓言
- **关联**：多模块协同架构的工程思路有启发

### CTRL (Keskar et al., 2019)
- **核心**：通过控制代码（control codes）控制文本生成属性
- **关联**：本项目的"大纲 + 状态卡"可视为叙事领域专门的控制代码

### PEER (Schick et al., 2022)
- **核心**：多模型协作写作（plan → write → edit）
- **关联**：plan-write 分离策略与本项目架构思路一致

## 二、记忆系统与上下文增强

### MemGPT (Packer et al., 2023)
- **核心**：LLM 操作系统隐喻——主存（核心上下文）、工作记忆（当前任务）、外存（长期记忆），通过函数调用管理
- **关联**：分层记忆思路直接启发了人物状态卡的双层设计（静态一次性建立 / 动态每回更新）
- **关键差异**：MemGPT 管理的是"对模型有用的信息"，本项目管理的是"必须遵守的硬约束"

### LongMem (Wang et al., 2023) — NeurIPS
- **核心**：通过 side-network 实现长程记忆的检索与更新
- **关联**：其"检索后注入上下文"的机制与"生成第 N 回前注入年表"的流程对应

### RAG (Lewis et al., 2020) — NeurIPS
- **核心**：检索增强生成，从外部知识库检索相关文档注入 prompt
- **局限性（针对本项目）**：基于语义相似度的检索无法区分"硬约束"（已逝人物不可出现）和"软参考"（可供参考的范文段落）
- **项目替代方案**：不依赖向量检索，而是基于规则（章节号）注入结构化年表 + 基于大纲人物列表注入对应状态卡

### StructLM (Zhuang et al., 2024)
- **核心**：探索将结构化表格数据注入 LLM
- **关联**：本项目年表和状态卡本质上是两张结构化表
- **增量**：本项目进一步定义了"不可逆"语义，区分硬约束与软参考

### Knowledge Neurons (Dai et al., 2022) — ACL
- **核心**：发现 Transformer FFN 层中存在存储事实性知识的神经元
- **关联**：解释了为什么微调后的模型仍"记得"前80回人物活跃状态——knowledge neurons 在 LoRA 微调时未被覆盖（LoRA 仅挂载 attention + FFN 投影层）

## 三、对齐与偏好优化

### DPO (Rafailov et al., 2023) — NeurIPS
- **核心**：直接偏好优化，无需显式训练 reward model，利用偏好对直接优化策略
- **项目中的应用（路径 A）**：利用规则型指标（ERA/CAC）自动构造 chosen/rejected 对，替代人工标注
- **优势**：叙事质量评估的人工标注成本极高，DPO + 自动化指标是实现闭环对齐的关键

### RLHF (Ouyang et al., 2022)
- **差异**：需训练 reward model 再做 PPO 优化，两步流程对叙事场景成本过高

### Constitutional AI (Bai et al., 2022)
- **核心**：自生成偏好数据 + 规则约束进行对齐
- **关联**：思路与本项目"规则型指标驱动 DPO 数据构造"一致

### Curriculum Learning (Bengio et al., 2009) — ICML
- **核心**：从易到难排列训练样本
- **项目中的应用（路径 B）**：ERA → CAC → CTSS 逐步收紧约束，每阶段只优化最难通过的指标

## 四、结构化人物建模与叙事世界

### CBDB（中国历代人物传记资料库）
- **核心**：结构化字段记录 37万+ 历史人物的姓名、籍贯、官职、亲属、社会关系等
- **直接启发**：本项目人物状态卡的字段化设计范式
- **本项目的增量**：增加了叙事生成所需的动态字段（当前位置、处境情感、允许出现形式），这些在纯历史数据库中不需要

### Generative Agents (Park et al., 2023) — UIST
- **核心**：memory stream → retrieval → reflection → planning，产生可信的日常行为
- **结构对应**：年表 ~ 共享记忆；状态卡 ~ 个体记忆
- **关键差异**：Generative Agents 侧重"自发行为生成"，本项目侧重"约束条件下的生成"

### Social Simulacra (Park et al., 2022) — UIST
- **核心**：多智能体社会模拟，展示个体 state tracking 的重要性
- **关联**：66 个角色的状态卡协同维护，本质上是叙事世界的 state tracking 问题

## 五、评估指标

### Perplexity (PPL)
- **项目中用法**：主指标，验证风格习得程度（目标：≤ 基线的 60%）
- **注意**：滑窗 chunk 首部 tokens 处于无上文状态，需用 label_mask_prefix_len 屏蔽以避免 PPL 虚高

### n-gram Overlap / StyleOverlap
- **项目中用法**：改性为 StyleOverlap（按角色切分），区分"风格习得"与"内容复制"
- **关键洞察**：Instruct 微调后期 PPL↓ 但 4-gram↓，说明模型正用自己的话重述而非背诵

### 叙事一致性评估（本项目的核心贡献方向）
- **现状空白**：当前无针对长程叙事一致性的多维度自动化指标
- **本项目方案**：六项规则型指标（WSC/ERA/TOC/CTSS/CAC/RCR）在纯文本空间可计算

## 六、《红楼梦》相关研究

### 红学（Redology）三大分支
- **考证派**（胡适《红楼梦考证》1921；俞平伯《红楼梦辨》1923；周汝昌《红楼梦新证》1953）：作者、版本、脂批考据。本项目大纲以判词和脂批为命运锚点，直接受益于考证成果。
- **索隐派**：试图从文本破解历史隐喻，与项目关注点不同。
- **文学批评派**：聚焦文本本身的艺术价值分析。

### 续写史
- **程高本后40回**（高鹗续，1791/1792）：流传最广但被普遍认为偏离曹雪芹原意。本项目大纲与高鹗续本有系统性差异（如黛玉之死的性质、宝玉出家的方式等），详见 `outline/chapter_titles_comparison.md` 的三版对照表。
- **文人续书**：清代有 30+ 部续书，但均为人类创作，无计算方法。

### 《红楼梦》文本计算研究
- **虚词分析与作者识别**：高频虚词分布区分前80回/后40回作者，是计算文体学经典案例
- **人物关系网络分析**：共现网络 → 量化中心度 → 宝黛钗叙事枢纽地位
- **情感分析与主题建模**：判词/诗词/对话的情感曲线和主题聚类
- **与项目的差异**：上述均为**分析性**（发现已有模式），本项目是**生成性**（创造新文本）
