# 面向经典文学续写的结构化叙事生成框架——以《红楼梦》后40回为沙盒验证

---

## 一、摘要

经典文学作品的续写在计算叙事领域面临三重挑战：长程世界状态一致性难以维持、人物命运轨迹在生成过程中易偏离预设约束、以及风格习得与内容控制之间的张力。现有基于大语言模型（LLM）的文本生成方法在单一回合内能产出流畅文本，但在多章回、多人物、跨数百页的连贯叙事中，频繁出现人物"复活"、状态漂移、时序错乱等一致性坍塌问题。

本文提出一个面向经典文学续写的分层叙事框架，将续写任务分解为三个独立可迭代的层次：**大纲约束层**（全局情节骨架）、**世界状态层**（年表 + 人物状态卡，注入显式上下文）、**文风表达层**（LoRA 微调习得风格）。以《红楼梦》后40回续写为沙盒（sandbox），基于 Qwen3-8B + QLoRA 完成前80回文风习得，并构建了包含66个角色的人物状态卡库、80回原文大纲和后40回续写大纲。针对生成过程中的世界状态违规问题，定义了六项对齐指标（WSC/ERA/TOC/CTSS/CAC/RCR），设计了 Prompt 层显式约束 + 训练层 DPO 对齐的双层解法。该框架以纯文本空间运行，不依赖视觉模态，其分层架构使其可迁移至其他经典作品的续写任务。

## 二、研究背景与意义

### 2.1 背景

大语言模型在短文本生成上已展示出强大的语言能力，但在长程叙事（long-form narrative generation）领域面临核心瓶颈：当前的自回归生成范式缺乏对"世界状态"的显式建模。模型通过 next-token prediction 习得的是语料分布，而非叙事世界的内在因果律。

经典文学续写将这一瓶颈推到极致——它不仅要求单章回内文风贴近原著、情节推进合理，还要求跨数十个章回维持人物生死、位置、关系变化的全局一致性。任何一个局部违规（如已逝人物在后续章回中"复活"）都会在读者端产生刺眼的断裂感。

### 2.2 问题定义

本研究将经典文学续写分解为三个子问题：

1. **世界状态一致性问题（World State Consistency）**：生成过程中，人物状态（生死/位置/身份）、事件时序、物品归属等约束如何被持续遵守？当前 prompt-only 方法因上下文窗口有限且模型缺乏对"不可逆事件"的结构化认知，导致状态漂移。

2. **人物与风格对齐问题（Character & Style Alignment）**：如何在保持各角色个性化语言风格（如黛玉的含蓄试探 vs 凤姐的泼辣犀利）的同时，不牺牲全局叙事连贯性？全局 n-gram overlap 指标无法区分"风格记忆"与"内容记忆"。

3. **框架可迁移性问题（Framework Transferability）**：如何设计一个不绑定特定作品的管线，使同一套方法可以从《红楼梦》迁移至《三国演义》《水浒传》等其他经典？

### 2.3 研究意义

**学术意义**：提出面向长程叙事的分层架构，将"世界状态管理"显式化为年表 + 人物状态卡两个结构化组件，并定义可量化的对齐指标体系。这为 LLM 在叙事一致性方向的研究提供了可复现的评估框架。

**应用意义**：为文化遗产数字化传承提供工具链——完整的原著→洗数据→微调→大纲→续写→评估管线，降低经典续写的技术门槛。该框架亦可扩展至互动叙事、AI-Generated 故事生成等商业场景。

## 三、相关工作

### 3.1 LLM 在叙事生成中的应用

LLM 在故事生成领域已有大量探索，但多数工作聚焦于短篇（<2000 词）。**PlotMachines** (Rashkin et al., 2020) 通过大纲短语控制故事走向，展示了 outline-conditioned generation 的可行性，但其评估仅在短篇范围内，不考虑跨章节状态一致性。**DOC** (Yang et al., 2022) 提出 plan → draft → polish 分阶段策略，但规划层仅覆盖情节节点，不包含世界状态变更。**Re3** (Yang et al., 2023) 引入 recall → relate → revise 三步策略提升叙事一致性，但依赖预训练重排序器，不适用于缺乏人工标注的古典文学领域。**FableStudio** (Majumder et al., 2024) 以多智能体协作生成寓言，其多模块协同架构有启发，但寓言长度远小于章回小说。指令微调方面，**CTRL** (Keskar et al., 2019) 通过控制代码控制生成属性，**PEER** (Schick et al., 2022) 的 plan-write 分离策略与本项目的"大纲约束层 → 文风表达层"思路相似。

**关键空白**：上述方法无需处理"跨数十个章回、跨越数年叙事时间"的世界状态管理，而经典文学续写因时间跨度大、人物关系网密，恰恰要求这种持久的状态追踪。

### 3.2 记忆系统与上下文增强

上下文增强领域与本研究高度相关，因为续写的核心困难之一就是"如何让模型记住之前发生的事"。

**长程记忆方向**：**MemGPT** (Packer et al., 2023) 在 LLM 中引入分层记忆（主存/工作记忆/外存），利用函数调用管理上下文，其分层思路对人物状态卡的双层设计（静态/动态字段）有直接启发。**LongMem** (Wang et al., 2023) 的 side-network 记忆检索与更新机制，在架构上与本项目"每回注入年表 + 状态卡"的机制对应。

**检索增强生成（RAG）** (Lewis et al., 2020) 是当前主流上下文增强方法。然而其核心局限在于依赖"语义相关度"检索，而非"约束性注入"——模型需要的是"林黛玉在第90回已逝，因此本回不可让她说话"这种硬约束，而非"最相似的黛玉段落"。向量相似度无法区分"必须遵守的规则"和"可参考的风格范例"。

**结构化知识注入**：**StructLM** (Zhuang et al., 2024) 探索将结构化表格数据注入 LLM。本项目的"年表 + 状态卡"与此类似，但进一步定义了"不可逆"语义以区分硬约束与软参考。**Knowledge Neurons** (Dai et al., 2022) 的研究解释了模型为何"记住"原文人物状态——我们的观察与此一致，解法是通过显式约束覆盖隐式记忆。

### 3.3 对齐与偏好优化

对齐领域与本研究的关系体现在：续写中的"世界状态违规"本质上可以转化为偏好信号。

**DPO** (Rafailov et al., 2023) 的出现使无奖励模型的 RLHF 成为可能。本项目的训练层方案正是利用规则型指标替代 reward model，自动构造 chosen/rejected 偏好对进行对齐微调，避免了叙事质量评估的人工标注瓶颈。**Constitutional AI** (Bai et al., 2022) 的自生成偏好数据 + 规则约束思路与此一致。**Curriculum Learning** (Bengio et al., 2009) 启发本项目的路径 B——从易到难收紧约束（ERA → CAC → CTSS），避免训练信号混淆。

**关键空白**：DPO/RLHF 目前主要用于安全对齐和指令遵循，在叙事一致性方向的系统性应用几乎为空白。本研究试图填补这一交叉领域。

### 3.4 结构化人物建模与叙事世界状态

**CBDB（中国历代人物传记资料库）** 以字段化方式记录了超过 37 万历史人物的籍贯、官职、亲属和社会关系，是结构化人物建模的标杆。本项目人物状态卡的字段设计受其启发，但增加了动态字段（当前位置、处境情感、允许出现形式）以满足叙事生成需求。

**Generative Agents** (Park et al., 2023) 为 AI 个体赋予记忆流（memory stream），通过 retrieval → reflection → planning 产生可信行为。其记忆架构（短期/长期/反思）与本研究的"年表（共享记忆）+ 状态卡（个体记忆）"有结构对应，但本研究侧重**约束生成**而非自发行为。**Social Simulacra** (Park et al., 2022) 进一步展示了个体 state tracking 对多角色场景的价值，与本项目维护 66 个角色状态一致性的需求共振。

传统游戏 NPC 系统使用有限状态机和行为树管理角色，提供了工程上成熟的状态管理范式，但其状态空间较小且规则为手工编写，无法直接适配章回小说中自然语言状态的复杂性。

### 3.5 《红楼梦》相关研究（红学与文本计算）

本项目以《红楼梦》为沙盒，需要充分了解红学和《红楼梦》文本计算领域的既有工作。

**红学（Redology）** 作为专学已有二百余年历史，主要分支包括：（1）**考证派**——以胡适、俞平伯、周汝昌为代表，聚焦作者、版本、脂批等历史考据问题；（2）**索隐派**——试图从文本中破解历史隐喻；（3）**文学批评派**——聚焦文本本身的艺术价值。本项目以判词、脂批和学界考证为依据构建后40回续写大纲，直接受益于红学考证成果。**《红楼梦》续写史**上，高鹗续本（程高本）是流传最广的 120 回版本，但其后40回在情节走向和人物命运上被普遍认为偏离了曹雪芹原意。此外还有大量文人续书（如《后红楼梦》《红楼复梦》等），但均为人类创作，尚无基于计算方法的系统性续写。

**《红楼梦》文本计算**：近年来计算文学方法在《红楼梦》研究中的应用主要包括——（a）**虚词分析与作者识别**：利用高频虚词分布进行前80回与后40回作者归属判定，是计算文体学（stylometry）的经典案例；（b）**人物关系网络分析**：构建人物共现网络，量化角色中心度与叙事结构，揭示了宝黛钗三人的叙事枢纽地位；（c）**情感分析与主题建模**：对判词、诗词、对话进行情感曲线和主题聚类分析。这些工作为本项目的人物状态卡字段设计和关系数据建模提供了丰富的前人成果可复用，但它们都是**分析性**的（发现文本中已有的模式），而非**生成性**的（创造新文本）。

**AI 续写《红楼梦》的现有尝试**多为单章生成 demo，利用通用 LLM 直接续写，没有针对文风做专门微调，也不考虑跨章回的世界状态一致性。这些尝试更多是展示 LLM 能力的"新鲜体验"，而非系统的学术探索。

### 3.6 小结：研究空白

| 方向 | 现有工作优势 | 与本研究直接相关的空白 |
|------|-------------|----------------------|
| LLM 叙事生成 | 短篇流畅、outline-condition 可行 | 跨多章回的世界状态一致性未解决 |
| 记忆系统 | MemGPT/RAG 提供上下文增强 | 缺乏"硬约束 vs 软参考"的区分机制 |
| 对齐优化 | DPO 降低偏好数据构造门槛 | 叙事一致性的自动化偏好对构造尚属空白 |
| 人物建模 | Generative Agents 提供记忆流框架 | 未处理"不可逆事件"作为叙事约束 |
| 评估指标 | PPL/n-gram 提供基础信号 | 缺乏针对叙事一致性的多维度自动化指标 |
| 红楼梦研究 | 文本挖掘和红学分析成果丰富 | **生成端几乎没有系统性工作** |

本项目的定位正是集合上述各方向的优势，针对这些空白提出一个完整的分层框架。

## 四、研究目标与核心问题

### 4.1 总目标

构建一个分层、可度量、可迁移的经典文学续写框架，使其能够在给定前 N 回原文和后 M 回大纲的条件下，生成风格一致、情节连贯、世界状态准确的续写文本。

### 4.2 研究问题

1. **RQ1（状态注入）**：如何以结构化方式（年表 + 状态卡）向 LLM 注入"截至本回为止世界是什么状态"的信息，使其在生成时遵守不可逆事件的约束？

2. **RQ2（对齐度量）**：如何定义一套仅在纯文本空间可计算的指标，量化生成文本与世界状态、人物设定的偏差程度？

3. **RQ3（训练对齐）**：如何利用对齐指标自动构造训练信号，从权重层面降低模型违规的先验概率？

4. **RQ4（框架迁移）**：当前以《红楼梦》为沙盒的管线中，哪些组件是作品特定的，哪些组件可以在最小改动下迁移至其他经典？

## 五、研究方法与技术路线

### 5.1 总体框架：三层模型

```
┌────────────────────────────────────────────┐
│  大纲约束层（Outline Layer）                │
│  · 全局情节骨架（后40回大纲）              │
│  · 判词/批语作为命运锚点                    │
│  · 职责：决定"发生什么"                     │
└───────────────┬────────────────────────────┘
                │ 注入
┌───────────────▼────────────────────────────┐
│  世界状态层（World State Layer）            │
│  · 年表（Timeline）：不可逆事件记录         │
│  · 人物状态卡（Persona）：动态人物属性      │
│  · 一致性校验规则                           │
│  · 职责：约束"在什么条件下发生"             │
└───────────────┬────────────────────────────┘
                │ 注入
┌───────────────▼────────────────────────────┐
│  文风表达层（Style Generation Layer）        │
│  · Qwen3-8B + QLoRA 文风微调               │
│  · Instruct 模式按大纲逐回生成               │
│  · 分场景多轮生成（scenes mode）            │
│  · 职责：决定"如何被叙述"                   │
└────────────────────────────────────────────┘
```

三层各自独立可迭代：大纲不涉及模型、状态层不依赖特定模型、表达层可替换不同基座模型。

### 5.2 核心模块

#### A. 数据预处理管线

- 文本清洗：去除现代注释、页面标记、统一标点
- 章回切分：按"第X回"正则切分为80个独立文件
- Tokenize + 滑窗分块：max_len=2048, stride=512，生成 train.jsonl / val.jsonl
- 训练/验证按章回划分（1-72回训练，73-80回验证），防止滑窗重叠导致 PPL 虚低

#### B. 文风微调

- 基座模型：Qwen3-8B，4-bit NF4 量化（bitsandbytes）
- 微调方式：QLoRA，r=16, alpha=32，仅对 attention + FFN 投影层插入适配器
- 训练目标：标准 Causal LM 自回归损失
- PPL 验证：从基线 23.84 降至 13.03（step 75）；Instruct 模式降至 7.52（step 75）

#### C. 大纲驱动生成

- 后40回大纲：基于判词、脂批、学界考证编写，四段弧线结构
- 每回大纲包含：核心情节、主要人物、关键场景、情感基调、叙事功能
- 生成方式：Instruct 模式，system prompt 注入文风要求，user prompt 注入大纲 + 前回结尾 + 人物状态卡
- 支持分场景多轮生成（--scenes），提高单回长度和细节控制

#### D. 世界状态管理（Prompt 层）

- **年表（Timeline）**：以回目为时间单位，记录不可逆状态变更（死亡、远嫁、出家、抄家等），含叙事纪时、事件摘要、影响人物、事件性质字段
- **人物状态卡（Persona）**：静态字段（姓名、出身、性格） + 动态字段（当前状态、所在位置、处境情感、关系变化、允许出现形式）
- **注入策略**：生成第 N 回时，注入第 81 至 N-1 回年表 + 本回出场人物状态卡
- **一致性校验**：以年表为准，在生成前检查状态卡与年表的一致性

#### E. 对齐指标体系（训练层）

六项仅在纯文本空间可计算的指标：

| 指标 | 缩写 | 对齐对象 | 计算方式 |
|------|------|----------|----------|
| 世界状态一致性分 | WSC | Timeline | 规则 + 模板匹配 |
| 事件引用准确率 | ERA | Timeline | 规则比对 |
| 时序一致率 | TOC | Timeline | 规则比对 |
| 人物文本风格相似度 | CTSS | Persona | 按角色切分的 n-gram overlap |
| 人物属性覆盖率 | CAC | Persona | 规则检查每个动态字段 |
| 人际关系一致率 | RCR | Persona | 规则检查关系字段 |

三项训练对齐路径：
- **路径 A（近期）**：利用规则型指标自动构造 DPO 偏好对（chosen/rejected），进行对齐微调
- **路径 B（中期）**：课程式约束收紧——先 ERA→1，再 CAC，最后 CTSS
- **路径 C（长期）**：将 CTSS 按角色拆分替代全局 n-gram 作为风格指标，实现风格与内容指标的解耦

|------|------|------|

## 六、当前进展与关键发现

### 6.1 风格习得验证

QLoRA 在训练75步（约3个 epoch）后，验证集 PPL 从 23.84 降至 13.03（降至基线的 54.6%），4-gram 重叠率从 0.13 升至 0.40。Instruct 模式在 75 步后 PPL 降至 7.52，生成文本在单章回内可以稳定产出接近原著风格的中文叙事。

关键发现：PPL 继续下降而 4-gram 在下降（从 0.43 降至 0.21），这表明 Instruct 微调正在迫使模型**用自己的话重述内容**（降低与训练集字面重叠）而非退化至原文记忆。

### 6.2 已识别的问题

1. **"林黛玉复活"问题**：在第81回之后的生成中，已按大纲死亡的人物（如黛玉第90回逝）在后续章回中频繁以对话发起者身份出现。根因：训练阶段以 n-gram overlap 和 perplexity 为指标，奖励"贴近训练语料"的输出，模型对前80回人物状态有极强的隐式记忆。当前 prompt 只含"将要发生什么"（大纲）和"刚发生了什么"（前文结尾），缺失"世界当前什么状态"的信息层。

2. **风格-内容张力**：PPL 持续下降但 4-gram 在某些 checkpoint 出现回涨，说明模型在"模仿原文"和"创造新内容"之间存在振荡。需要更精细的指标来区分风格习得和内容复制。

3. **多回连续性衰减**：单回生成质量随回目数增加而波动，后续回目的 bug（如人物复活、情节错乱）需要向前追溯修正，而非局部修复。

### 6.3 待验证假设

- 注入年表 + 状态卡后，人物"复活"类违规的发生率能否显著降低？
- DPO 对齐能否在不损害风格质量的前提下，将 WSC 和 ERA 提升至可接受水平？
- 该框架的关键抽象（大纲→状态层→表达层的三段式）是否足够通用，可以零修改或少量修改迁移至其他古典小说？

## 七、创新点

1. **分层解耦的经典续写架构**：将续写任务分解为大纲约束、世界状态管理、文风表达三个独立层次，每层有独立的输入/输出接口和评估标准，允许独立优化和替换。

2. **不可逆事件的结构化建模**：引入年表（Timeline）作为显式世界状态表示，以"不可逆性"为筛选标准管理全局约束，并通过一致性校验规则防止生成违规。这一设计源自传统史书体例（《史记·十二诸侯年表》），但在 LLM 叙事生成中尚无类似工作。

3. **文本空间对齐指标体系**：定义六项仅在纯文本空间可计算的指标（WSC/ERA/TOC/CTSS/CAC/RCR），无需额外标注或训练分类器，使得生成质量评估和 DPO 偏好对构造可以全程自动化。

4. **从工程系统抽象研究问题**（methodology innovation）：当前工作不是从零假设出发进行实验设计，而是从真实 Pipeline 中观察违规现象、分析根因、设计解法。这种"debug 驱动"的研究范式为 LLM 叙事生成领域提供了可复现的问题定义和评估基准。

5. **可迁移框架设计**：以《红楼梦》为 sandbox 验证核心假设，但框架设计时已将作品特定部分（原文、大纲、人物）与通用部分（状态管理、对齐指标、训练管线）明确分离，为迁移至其他经典做好了架构准备。

## 十一、当前挑战与待解决问题

1. **年表构建自动化**：当前年表为手工编写，需探索如何自动化抽取每个生成回目的不可逆事件。
2. **状态卡更新机制**：动态字段（当前位置、处境情感、关系变化）需每回生成后更新，手工维护成本高，需设计半自动或 LLM 辅助更新流程。
3. **评估标准的主观性**：CTSS 等风格指标涉及审美判断，人工评估的标准化与一致性是挑战。
4. **框架迁移验证**：需要在至少一部其他经典作品上完整走通管线，以证明框架的可迁移性。
5. **极长程一致性**：120回规模的叙事一致性，当前未见同类工作可参照，评估标准需要从第一性原理出发定义。
