# 文本生成与翻译评估指标示例

本文列举 BLEU、F-Measure、NIST、ROUGE、WER、LEPOR、Noun-Phrase Chunking、METEOR 等常见指标的用途与简化示例。示例以自然语言生成、机器翻译、摘要和语音识别评估为主。

---

## 统一示例

**参考文本 Reference**

> 贾宝玉走进园中，看见林黛玉正在葬花。

**候选文本 Candidate A**

> 贾宝玉走进园子，看见林黛玉在葬花。

**候选文本 Candidate B**

> 林黛玉走进园中，看见贾宝玉正在读书。

直观上，Candidate A 与参考文本更接近：人物、动作和场景基本一致，只是个别词不同；Candidate B 虽然也包含部分人物和地点词，但事件关系已经变了。

---

## 1. BLEU

**适用场景**：机器翻译、文本生成。
**核心思想**：比较候选文本和参考文本之间的 n-gram 重合率，并加入 brevity penalty 避免过短输出得高分。

**简化例子**

参考文本：`贾宝玉 / 走进 / 园中 / 看见 / 林黛玉 / 正在 / 葬花`

Candidate A：`贾宝玉 / 走进 / 园子 / 看见 / 林黛玉 / 在 / 葬花`

1-gram 中，`贾宝玉、走进、看见、林黛玉、葬花` 都匹配，得分较高。
2-gram 中，`贾宝玉 走进`、`看见 林黛玉` 等也能匹配一部分。
因此 BLEU 会认为 Candidate A 质量较好。

**注意**：BLEU 偏重字面重合，不擅长识别语义等价但措辞完全不同的表达。

---

## 2. F-Measure

**适用场景**：信息抽取、分词、分类、问答、实体识别等。
**核心思想**：综合 Precision 和 Recall。

公式：

```text
Precision = 正确预测数 / 预测总数
Recall = 正确预测数 / 标准答案总数
F1 = 2 * Precision * Recall / (Precision + Recall)
```

**简化例子：人物名识别**

标准答案人物：`贾宝玉、林黛玉、薛宝钗`

系统预测人物：`贾宝玉、林黛玉、王熙凤`

正确预测：`贾宝玉、林黛玉`

```text
Precision = 2 / 3 = 0.667
Recall = 2 / 3 = 0.667
F1 = 0.667
```

**注意**：F-Measure 本身不是文本生成专用指标，但常用于评估生成文本中的实体、事实点、关键词是否覆盖正确。

## 3. NIST

**适用场景**：机器翻译。
**核心思想**：和 BLEU 一样基于 n-gram 匹配，但会给信息量更高、较少见的 n-gram 更大权重。

**简化例子**

参考文本：

> 贾宝玉 梦游 太虚幻境

候选 1：

> 贾宝玉 梦游 太虚幻境

候选 2：

> 贾宝玉 来到 一个地方

两者都匹配了 `贾宝玉`，但候选 1 还匹配了 `梦游 太虚幻境` 这种更具体、更有信息量的片段。NIST 会更奖励候选 1。

**注意**：NIST 相比 BLEU 更关注“匹配到了多少有信息的内容”，而不是简单平均 n-gram 精度。

---

## 4. ROUGE

**适用场景**：自动摘要、长文本生成。
**核心思想**：主要从 Recall 角度衡量候选文本覆盖了多少参考文本内容。

常见变体：

- **ROUGE-1**：单词或字的重合。
- **ROUGE-2**：二元组重合。
- **ROUGE-L**：最长公共子序列。

**简化例子**

参考摘要：

> 宝玉入园，见黛玉葬花，二人因花落而伤感。

候选摘要：

> 宝玉在园中见黛玉葬花。

候选摘要覆盖了 `宝玉、园中、黛玉、葬花` 等核心信息，但漏掉了 `二人伤感`。ROUGE 会给它中等偏高分，而不是满分。

**注意**：ROUGE 适合看内容覆盖率，但可能高估堆砌关键词的摘要。

---

## 5. Word Error Rate (WER)

**适用场景**：语音识别、OCR、转写系统。
**核心思想**：计算从候选文本变成参考文本需要多少次插入、删除、替换操作。

公式：

```text
WER = (S + D + I) / N
```

其中：

- `S` = substitutions，替换数
- `D` = deletions，删除数
- `I` = insertions，插入数
- `N` = 参考文本词数

**简化例子**

参考文本：

> 贾宝玉 看见 林黛玉 葬花

识别结果：

> 贾宝玉 看见 林黛玉 种花

只有 `葬花` 被替换成了 `种花`。

```text
S = 1, D = 0, I = 0, N = 4
WER = 1 / 4 = 0.25
```

**注意**：WER 越低越好。它对语义不敏感，只关心编辑距离。

## 6. LEPOR

**适用场景**：机器翻译。
**核心思想**：综合长度惩罚、词序惩罚、Precision 和 Recall。

LEPOR 的名字来自几个核心因素：

- **Length Penalty**：长度是否接近参考文本。
- **Position Difference Penalty**：词序是否接近。
- **Precision**：候选文本中有多少内容正确。
- **Recall**：参考文本中有多少内容被覆盖。

**简化例子**

参考文本：

> 宝玉 在 园中 看见 黛玉

候选 1：

> 宝玉 在 园中 看见 黛玉

候选 2：

> 黛玉 看见 园中 宝玉

候选 2 的词大多出现了，但顺序错乱，含义也可能改变。BLEU 和 ROUGE 可能仍给部分重合分，而 LEPOR 会通过词序惩罚降低候选 2 的得分。

**注意**：LEPOR 比单纯 n-gram 重合更重视长度、召回、精度与词序的平衡。

## 7. Noun-Phrase Chunking

**适用场景**：浅层句法分析、信息抽取、文本结构分析。
**核心思想**：识别名词短语块，例如人物名、地点名、物品名、修饰性名词短语。

**简化例子**

句子：

> 年轻的贾宝玉走进大观园。

可能的名词短语块：

```text
[年轻的贾宝玉] 走进 [大观园]
```

如果标准答案是：

```text
[年轻的贾宝玉], [大观园]
```

系统输出是：

```text
[贾宝玉], [大观园]
```

系统识别出了核心人物和地点，但漏掉了修饰语 `年轻的`。可以用 Precision、Recall、F1 来评估 chunk 边界和内容是否正确。

**注意**：Noun-Phrase Chunking 不是生成质量指标本身，但可作为生成文本的结构分析工具，例如检查人物、地点、物品短语是否完整。

---

## 8. METEOR

**适用场景**：机器翻译、文本生成。
**核心思想**：在词匹配基础上考虑词形变化、同义词、词序片段，并综合 Precision 与 Recall。相比 BLEU，METEOR 通常更接近人工评价。

**简化例子**

参考文本：

> 黛玉 悲伤地 葬花

候选 1：

> 黛玉 伤心地 埋花

候选 2：

> 黛玉 高兴地 赏花

如果系统有同义词或近义词资源，`悲伤地` 和 `伤心地`、`葬花` 和 `埋花` 可以被部分匹配。因此 METEOR 可能认为候选 1 比候选 2 更好，即使两者与参考文本的字面重合都不完全。

**注意**：METEOR 的优势在于比 BLEU 更能处理同义表达，但依赖语言资源和匹配规则。

---

## 9. 指标对比小结

| 指标 | 常见用途 | 越高越好 | 主要关注点 |
| --- | --- | --- | --- |
| BLEU | 机器翻译、生成 | 是 | n-gram 精度 |
| F-Measure | 抽取、分类、实体评估 | 是 | Precision 与 Recall 平衡 |
| NIST | 机器翻译 | 是 | 信息量加权的 n-gram 匹配 |
| ROUGE | 摘要、长文本生成 | 是 | 内容覆盖率 |
| WER | 语音识别、转写 | 否 | 编辑错误率 |
| LEPOR | 机器翻译 | 是 | 长度、词序、精度、召回 |
| Noun-Phrase Chunking | 句法分析、信息抽取 | 通常是 | 名词短语边界和内容 |
| METEOR | 机器翻译、生成 | 是 | 词形、同义词、词序和召回 |

---

## 10. 选择建议

- 评估机器翻译：优先考虑 BLEU、METEOR、NIST、LEPOR。
- 评估摘要：优先考虑 ROUGE，必要时结合人工评价。
- 评估语音识别或转写：使用 WER。
- 评估实体、关键词、名词短语是否正确：使用 Precision、Recall、F1，或结合 Noun-Phrase Chunking。
- 评估文学续写或叙事生成：这些指标只能衡量表层重合或局部结构，仍需要补充人物一致性、情节一致性、风格相似度和人工评价。
